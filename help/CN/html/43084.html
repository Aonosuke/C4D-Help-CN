渲染队列

开始渲染命令在任务菜单中。

如果你之前在使用 Batch 渲染器，会发现新的渲染队列与之相似，但是提供更多的功能，而且使用更加方便。

例如：

  • 你可以连续渲染几乎任何数量的文件而不需要监视
  • 你可以轻易的修改保存路径、选择摄像机、场次和渲染设置，而不需要打开文件本身
  • 渲染队列配置可以保存为预设
  • 日志文件可以保存和查看
  • 渲染图像会在渲染过程中显示在预览窗口中
  • 动画渲染（单独帧，而不是视频）可以中断，而之后可以在同一个路径下自动继续渲染
  • 可以在图片查看器和渲染队列中同时进行渲染
  • 场景不需要在视窗中打开就可以渲染，这样可以节约内存

列表管理器顶部显示了任务列表，场景文件该如何添加到这个列表中？

  • 使用文件菜单命令
  • 从文件夹/内容浏览器中拖放

当一个文件在任务列表中时，第二栏的选框就可以用来启用或禁用渲染。多个任务可以使用热键（Shift：连续选择；Ctrl/Cmd：选择/取消选择单个任务）。任务可以移动到列表中的其他位置或删除。选择的任务可以按住Ctrl/Cmd键复制。

提示：
 注意，渲染队列只能使用保存的场景，在 CINEMA 4D 中打开的场景如果被修改过但没有保存则不可以使用。

提示2：
如果渲染中碰到计算机需要睡眠或注销的问题，即使 CINEMA 4D 还需要几个小时来渲染，只需要禁用“停止活动XX分钟后注销计算机”（Mac OS：系统偏好设置/安全设置）或者将注销时间设置到允许渲染完成，你还可以延长“唤醒时需要密码”的时间设置。

任务列表包含如下信息：

位置：所有任务是连续编号的。

渲染（是/否）：当选择了任务菜单中的开始渲染时，所有勾选了选框的任务会按照从上到下的顺序进行渲染。

团队：你可以在这里定义哪些任务使用团队渲染。

状态 R：显示任务当前的状态。

在队列中/R：准备渲染。

在进程中：正在被渲染。

场次：选择要渲染额场次（这个选择可以在窗口下方进一步设置）。

已完成：渲染已经成功完成。

已停止：渲染在完成前被停止。

错误：在渲染中发生错误。

摄像机：渲染所使用的摄像机，可以在渲染队列窗口的底部选择。

渲染设置：渲染的文件所使用的渲染设置，这也可以在渲染队列窗口底部进行选择。

信息：相关的信息会在这里显示，如文件丢失、保存信息等等。

一个已有的任务列表会保留，即使你关闭了 CINEMA 4D。这个列表会在 CINEMA 4D 重新启动并打开渲染队列时再次显示。

在渲染队列窗口的底部你会找到选中任务的相关信息、渲染的最后一帧图像预览、以及下列设置和按钮：

RELEASE 17

场次

这个选择菜单显示选择的任务文件中包含的所有场次，场次中定义的渲染摄像机和渲染设置会通过下列两个选择菜单自动设置。

渲染设置

这个下拉列表包含了选中任务的所有渲染设置，选择你在渲染任务中要使用的渲染设置。默认情况下会使用场景中激活的渲染设置。

摄像机

这个下拉列表包含了选中任务和舞台对象的所有摄像机（包括透视视图），选择你想用的摄像机来渲染场景。如果选择了一个舞台对象，那么会使用它的动画摄像机。

输出文件

你可以在这里设置一个保存路径，它可以与选中任务场景中设置的路径不同。如果你不选择一个不同的路径，则会使用场景中定义的路径。渲染队列管理器会检查准确的保存路径和文件名是否被其他任务使用，并且会自动修改文件名（防止一个任务覆盖另一个）。

提示：
如果要根据项目名称或渲染分辨率来指定文件名和路径名，从而不需要每次更改分辨率后更改文件名/路径名，且不会覆盖现有的文件，那么可以利用可变路径和文件名设置。

多通道文件

如果在场景中定义了多通道文件（或者多通道保存路径），则可以在这里输入多通道文件保存路径。

日志

渲染队列管理器会为每个任务进程创建一个日志文件，在这里可以设置它的保存路径。这个日志会显示在日志文件标签页中，即使日志本身没有保存。可以在程序设置中禁用日志文件的保存功能。

纹理错误

如果一个任务场景丢失了纹理，则会在这里显示它们的名称。

日志文件标签

与选择的渲染任务、系统、场景以及渲染结果相关的信息会显示在这里。在 显示/过滤 菜单中，你可以选择要显示哪些日志信息。

在设置…菜单的渲染器设置中，你可以定义是否总是保存日志文件，否则这个文件只会临时保存——只要该任务还保留在任务列表中。

更新按钮

这个按钮可以在渲染进行时使用，用来刷新显示最新的日志信息。

文件菜单

打开…

你可以从这个菜单中打开任何 CINEMA 4D 文件，它会被添加到任务队列。你还可以直接从文件夹中将文件拖放到任务队列中。

添加当前工程

这个命令会加载当前视窗中的场景到任务列表中，它也可以在主界面的渲染菜单中单独使用。

RELEASE 17

打开所有场次
打开已标记的场次

使用这两个命令可以打开一个工程的所有或标记的场次，它们会作为单独的任务加入到渲染队列中。如果没有场次被标记，则会使用主要场次。

提示：
场次管理器的渲染菜单也可以用来渲染多个场次，而不需要使用渲染队列（参考渲染到图片查看器）。

为所有场次添加工程
为标记的场次添加工程

使用这两个命令可以将当前打开场景的所有或标记的场次作为单独的任务添加到渲染队列中。如果没有场次被标记，则会使用主要场次。

撤销保存

如果你修改了渲染队列管理器下方的路径、摄像机或渲染设置，这个命令可以重设相应的元素到它在场景中保存的状态。

载入预设…
保存预设…

渲染队列配置可以作为预设保存在内容浏览器中（Presets/User/Queue Manager Preset），你可以在任何时候使用它们。

任务菜单

开始渲染

依次开始所有激活任务的渲染进程。

停止渲染

停止渲染进程。进程可以用前一个命令重新开始。部分地渲染的影片会再次从头开始渲染（与单独渲染图片相反，后者会从停止处继续渲染任务）。

删除

从任务列表中删除所有选择的任务。

清除全部

删除任务列表中所有的任务。

编辑工程

加载所有选择的场景到 CINEMA 4D 中，然后就可以编辑它们了。别忘了保存场景，这样渲染队列管理器会渲染它们最新的版本。

纹理错误

如果启用这个选项，丢失了纹理的渲染任务（仅有这些）会停止，队列管理器会自动继续下一个任务。如果禁用，无论丢失任何纹理，任务都会被渲染。

检查纹理

使用这个选项可以在加载新任务时检查是否丢失纹理（如果启用了这个选项，则会在任务列表中出现一个错误信息）。查找纹理可能会花费很长时间，因此可以禁用这个选项，这时工程仍然会被渲染。

设置…

打开设置的渲染器设置页。

显示/过滤

在图片查看器中打开…

在图片查看器中显示文成的图片/图片序列。

在浏览器/查找器中打开…

打开渲染的图片所在的文件夹。

在搜索引擎/浏览器中打开多通道文件

打开多通道文件所在的文件夹。

打开日志

在系统默认浏览器中打开选中任务的日志文件（与日志文件标签页中显示的一样）。

过滤日志

系统
渲染
工程

你可以在这里设置在日志文件标签页中显示哪些信息。系统信息一般不太重要，可以不用显示它。
